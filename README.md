## 🎮 Usage & Configuration

### Automatic Optimization
The mod works automatically upon installation. No manual configuration is required for basic use. Optimizations include:

- **FPS Optimization**: Automatically adjusts intensity based on system performance
- **Render Distance**: Intelligently manages extended render distances (up to 512 chunks)
- **Memory Management**: Automatic cleanup and caching
- **Entity Optimization**: Smart culling based on distance and player view direction
- **Network Optimization**: Reduces bandwidth usage in multiplayer

### Configuration Options
The mod includes configuration files in `config/ultimateoptimization-common.toml` and `config/ultimateoptimization-client.toml` where you can customize:

```toml
# FPS Settings
fpsBoostIntensity = 25  # 1-25 (higher = more aggressive)
targetFPS = 60

# Render Distance
maxRenderDistance = 512  # 2-512 chunks
extendedDistanceEnabled = true

# Entity Optimization
entityRenderDistance = 96  # 64-200 blocks
entityOptimizationEnabled = true

# Memory Management
aggressiveMemoryCleanup = false
maxCacheSize = 1500
memoryPressureThreshold = 75.0

# Multiplayer Settings
multiplayerOptimization = true
playerTrackingDistance = 128
```

## 🐛 Known Issues & Troubleshooting

### Common Issues:

1. **Mod Conflicts:**
   - If using other performance mods, disable overlapping features
   - Disable embedded optimizations if using Sodium/Embeddium

2. **Memory Issues:**
   - Reduce `maxCacheSize` in config if experiencing memory pressure
   - Enable `aggressiveMemoryCleanup` if using low RAM systems

3. **Stuttering:**
   - Reduce `fpsBoostIntensity` below 25 if experiencing stutter
   - Disable extended render distance if GPU is bottleneck

4. **Multiplayer Issues:**
   - Some servers may block certain optimizations
   - Reduce `playerTrackingDistance` if experiencing lag
